# Change: 优化职业五维画像参数

## Why

当前职业数据库存在三个主要问题，影响匹配算法的准确性和用户体验：

1. **评分范围过窄**：职业要求最高才0.7，但优秀用户可达1.85（超出2.6倍），导致需要"方向性匹配"算法补偿，治标不治本。
2. **五维使用不足**：78%的职业只用3维（O, C, N），忽略了外向性和宜人性对社交型、共情型职业的重要性。
3. **硬性门槛缺失**：仅11%的职业有最低要求，导致不适合的高风险职业可能被推荐给能力不足的用户。

这些问题导致：
- 高分用户（O>1.0, N>1.0）的匹配分数偏低
- 社交型职业（教师、HR）无法筛选出外向的候选人
- 高风险职业（医生、咨询师）可能推荐给情绪不稳定的用户

## What Changes

### 1. 扩大评分范围（高优先级）
- 将核心职业的 O, C, N 要求从 0.0-0.7 范围扩展到 -0.5 ~ 1.5
- 调整策略：
  - 创意型职业（艺术总监、科研人员）：O 提升至 1.2-1.5
  - 高压职业（医生、投资）：N 提升至 0.8-1.0
  - 严谨型职业（财务、审计）：C 提升至 0.8-1.0
- 影响：全部37个职业

### 2. 增加硬性门槛（高优先级）
- 新增门槛的职业（10个）：
  - 临床医生：C ≥ 0.3, N ≥ 0.3
  - 心理咨询师：N ≥ 0.4, A ≥ 0.2
  - 投资分析师：N ≥ 0.3
  - 财务会计：C ≥ 0.4
  - 科研人员：O ≥ 0.3
  - 管理咨询顾问：C ≥ 0.2, N ≥ 0.2
  - 学科教师：A ≥ 0.1
  - 社会工作者：A ≥ 0.2, N ≥ 0.2
  - 销售经理：E ≥ 0.2
  - 在线教育讲师：E ≥ 0.1

### 3. 启用外向性维度（中优先级）
- 新增外向性的职业（15个）：
  - 市场营销经理：E = 0.4
  - 人力资源经理：E = 0.3
  - 产品经理：E = 0.2
  - 学科教师：E = 0.2
  - 临床医生：E = 0.1
  - 保险规划顾问：E = 0.4
  - 企业培训师：E = 0.3
  - 管理咨询顾问：E = 0.3
  - 品牌视觉设计师：E = 0.1
  - 内容创作者：E = 0.0
  - 艺术总监：E = 0.2
  - 心理咨询师：E = 0.2
  - 社会工作者：E = 0.2
  - 公益项目经理：E = 0.3
  - UX 设计师：E = 0.2

### 4. 启用宜人性维度（中优先级）
- 新增宜人性的职业（10个）：
  - 心理咨询师：A = 0.5
  - 社会工作者：A = 0.5
  - 临床医生：A = 0.3
  - 学科教师：A = 0.4
  - 人力资源经理：A = 0.4
  - 企业培训师：A = 0.3
  - 健康管理师：A = 0.3
  - 公益项目经理：A = 0.4
  - UX 设计师：A = 0.2
  - 产品经理：A = 0.2

## Impact

### Affected Specs
- `career-matching`：职业画像参数调整
- 可能影响 `norm-system`：未来可能需要扩展常模数据

### Affected Code
- `tpl/seed_occupation.json`：职业数据库（主要修改）
- `src/apps/begreat/miniapp/services/MatchingService.ts`：已支持5维，无需修改
- `test/check_occupation_dimensions.ts`：验证脚本

### Affected Data
- 职业数据：37个职业的 requiredBig5 和 minimumRequirements 字段

### Expected Outcomes
1. **高分用户体验改善**：O>1.0, N>1.0 的用户匹配分数从60-70提升至75-85
2. **匹配精度提升**：社交型职业筛选出外向候选人，共情型职业筛选出宜人候选人
3. **安全性增强**：高风险职业不会推荐给能力不足的用户
4. **减少算法补偿**：方向性匹配（0.5倍惩罚）的作用减弱，更依赖真实匹配度

### Risks
- **现有用户结果变化**：已完成测评的用户重新计算后，推荐结果可能改变
- **数据校准工作量**：需要逐个职业调整参数，容易出错
- **缺乏真实数据验证**：基于经验估计，非实际从业者数据

### Mitigation
- 保留旧数据版本，支持 A/B 测试
- 分批验证：先优化10个核心职业，观察效果再扩展
- 记录调整理由：在代码注释或文档中说明每个参数的依据
